Axios

Axios 是一个基于 Promise 的 HTTP 客户端，能够在浏览器和 Node.js 中发起 HTTP 请求。它简化了与服务器的通信，支持各种请求方法（如 GET、POST、PUT 等）以及处理请求和响应的相关操作。下面是关于 Axios 的详细学习指南。

1. 安装 Axios

3. 常见请求方法

  • axios.get(url, config): 发起 GET 请求。
  • axios.post(url, data, config): 发起 POST 请求。
  • axios.put(url, data, config): 发起 PUT 请求。
  • axios.delete(url, config): 发起 DELETE 请求。

每种方法都会返回一个 Promise，可以通过 then 和 catch 处理响应和错误。

4. 配置请求（Config）

你可以通过 config 对象来自定义请求，常见的配置选项有：

  • url: 请求地址。
  • method: 请求方法（如 GET、POST）。
  • headers: 设置 HTTP 请求头。
  • params: 设置 URL 查询参数。
  • data: 发送到服务器的数据（适用于 POST、PUT 等方法）。
  • timeout: 请求超时时间。

问：async/await有什么用

async/await 是 ES2017 (ES8) 引入的 JavaScript 语法，用于处理异步操作，提供了更简单、更可读的方式来处理 Promises。相比于传统的 .then() 和 .catch() 链式处理，async/await 使异步代码的编写更加像同步代码，提高了代码的可读性和维护性。

1. 基本概念

  • **async**：用于定义一个异步函数，返回一个 Promise。异步函数允许在其中使用 await 关键字。
  • **await**：只能在 async 函数中使用，等待一个 Promise 完成，并暂停函数执行，直到 Promise 解决（resolve）或拒绝（reject）。

  • try...catch：用于处理可能的错误，代替 .catch()。

5. 注意事项

5.1 await 只能在 async 函数中使用

如果在非 async 函数中使用 await，会导致语法错误。

5.2 阻塞代码执行

虽然 async/await 看起来像同步代码，但它本质上还是异步的。await 会暂停函数的执行，直到 Promise 被解决（resolved），这意味着如果一个异步操作耗时较长，它会阻塞后续代码的执行。因此，使用时要注意避免长时间阻塞。

5.3 错误处理

尽量使用 try...catch 来捕获 await 操作中可能发生的错误。如果没有捕获，错误可能会被静默忽略，导致调试困难。

总结

  • 简洁性：async/await 使异步代码更加简洁直观，接近同步代码的结构，避免回调地狱。
  • 错误处理：可以使用 try...catch 统一处理异步操作中的错误。
  • 性能：与 Promise.all 结合，可以并发执行多个异步任务，提升性能。

通过 async/await，你可以更加清晰地组织异步代码，并提高代码的可维护性和可读性。
