Telegraph API 概述

Telegraph API 概述

Telegra.ph 是一个简约的发布工具，允许用户快速创建格式丰富的文章，并将其推送到网络上。Telegraph 文章在 Telegram 上也能生成美观的 Instant View 页面。

主要功能

  • @Telegraph Bot：用于管理文章和获取页面浏览统计信息。
  • 开放 API：任何开发者都可以利用 Telegraph API 创建类似 @Telegraph 的机器人，适用于其他平台或独立接口。

API 请求

所有 API 请求必须通过 HTTPS 进行，格式为 https://api.telegra.ph/%method%。

可用方法

  1. createAccount：创建新账户。
  2. createPage：创建新页面。
  3. editAccountInfo：更新账户信息。
  4. editPage：编辑现有页面。
  5. getAccountInfo：获取账户信息。
  6. getPage：获取指定页面。
  7. getPageList：获取账户下所有页面列表。
  8. getViews：获取文章的浏览次数。
  9. revokeAccessToken：撤销访问令牌。

数据格式

API 响应为 JSON 对象，包含 ok 字段以指示请求成功与否。

账户对象

  • short_name：账户名。
  • author_name：默认作者名。
  • author_url：作者链接。
  • access_token：账户的访问令牌。

页面对象

  • path：页面路径。
  • url：页面链接。
  • title：页面标题。
  • content：页面内容。

内容格式

结论

Telegraph API 提供了一个简单易用的接口，适合需要快速发布内容的用户和开发者。通过该 API，可以轻松管理和创建文章，适用于多种应用场景。

详细介绍

Telegra.ph 提供了一个简单的 API，允许用户通过程序化方式发布文章、获取文章信息等操作。以下是 Telegra.ph API 的总结，帮助你快速入门，并注意一些关键点。

Telegra.ph API 概览

  1. API 端点:

你需要注意的几点：

  1. Token 管理：

    • 每次创建帐户后会得到一个 access_token，这个 token 对于管理文章和帐户是必需的。确保保存并保护好它，因为它相当于你的身份认证。
    • 一旦丢失或删除浏览器缓存，可能会失去对发布文章的编辑权限。

  2. 文章内容的格式：

    • content 参数以 JSON 数组的形式发送，支持 HTML 标签如 <p>、<b> 等。要确保文章内容符合 API 规定的格式，避免因格式错误导致请求失败。

  3. 文章路径：

    • 每个发布的文章都有一个唯一的 path，比如 Sample-Page-12-10，它用于通过 API 获取、编辑或删除该页面。

  4. 限流与性能：

    • Telegra.ph API 是简洁的，但并没有详细说明关于限流等问题。确保遵循最佳实践，不频繁发送请求，以免被系统暂时屏蔽。

  5. 无登录的限制：

    • Telegra.ph 不需要登录账户，但是因为使用 access_token 管理帐户和页面，一旦遗失 token，可能无法恢复访问权限。

  6. 隐私和匿名性：

    • 发布的内容是公开的，没有用户隐私保障。平台允许匿名发布文章，但内容的访问是公开的，因此需要对内容的发布和分享保持谨慎。
